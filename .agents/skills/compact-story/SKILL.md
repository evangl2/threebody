# Compact-Story · 章节压缩

## 职责
将已完成且稳定的早期章节压缩为详细摘要。
保留所有故事关键信息，减少后续工作的上下文负担。
原始章节文件永久保留，不删除。

## 触发条件
- 章节数量超过十章
- 作者主动要求
- Meta Agent建议启动

## 压缩对象选择原则
适合压缩的章节：
- 已完成且不太可能被修改的早期章节
- 作者明确表示已定稿的章节

不适合压缩的章节：
- 最新五章（可能还在修改）
- 作者标注为"待修改"的章节
- 包含后续章节频繁引用的关键场景

## 压缩程序

### Step 1：确认压缩范围
与作者确认哪些章节可以压缩。

### Step 2：逐章提取关键信息

从原始章节文件提取：

```
━━━ 压缩提取清单 ━━━
场景与时间：
  - 故事时间点（年份/月份）
  - 地点
  - 历史事件背景

核心情节点：
  - 发生了什么（按时序列出）
  - 关键转折

重要对话要点：
  - 不记录原文，记录要点和意涵
  - 标注说话人

人物状态变化：
  - 本章中各人物的状态变化
  - 关系的微妙变化

情感弧线：
  - 本章的情感密度和走向
  - 关键情感时刻（描述，不引用原文）

女翻译视角的关键观察：
  - 她注意到了什么
  - 她的推测性描写要点
  - 她个人生活的相关内容

后续章节的引用点：
  - 本章哪些内容可能被后续章节引用
```

### Step 3：写入压缩摘要

摘要写入两个位置：

**位置A：story-index.md对应条目**
在章节条目下新增`#### 详细摘要`小节，
写入完整的压缩内容。
将`Compacted`字段改为`是`。

**位置B：原始章节文件顶部**
在原始章节文件最顶部添加标注：
```markdown
<!-- 已Compact
Compact日期：[日期]
摘要位置：/story/story-index.md ##第X章
原始内容保留在下方。
-->
```

### Step 4：验证摘要完整性

对照提取清单，确认摘要包含了所有关键信息：
```
✅ 场景与时间：完整/缺失
✅ 核心情节点：完整/缺失
✅ 重要对话要点：完整/缺失
✅ 人物状态变化：完整/缺失
✅ 情感弧线：完整/缺失
✅ 女翻译关键观察：完整/缺失
✅ 后续引用点：完整/缺失
```

若有缺失，补充后再标注为完成。

### Step 5：输出压缩报告

```
【章节压缩报告】
压缩章节：第X章至第X章（共X章）
压缩日期：
原始文件状态：已标注，内容保留
story-index更新：已完成

各章摘要字数：
  第X章：约XXX字
  第X章：约XXX字

注意事项：（如有特殊情况说明）
```

## 摘要质量标准
- 摘要必须让后续agent在不读原文的情况下
  完成人物一致性检验和情感弧线追踪
- 不引用原文段落，只描述要点
- 保留所有人名、地名、时间的准确信息
- 情感描述用简洁的叙事语言，不用文学语言

## 禁止事项
- 禁止删除原始章节文件
- 禁止修改原始章节文件的正文内容
- 禁止压缩未获作者确认的章节
