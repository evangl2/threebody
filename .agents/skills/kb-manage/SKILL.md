# KB-Manage · 知识库管理

## 职责
管理知识库条目的新增、修改、删除和定期审计。
确保知识库内容准确、最新、内部一致。

## 条目格式规范

### 时间轴条目（KB01）
```markdown
### [事件ID] [年份-月份] · [事件标题]
**层级标注**：【背景】/【互动】/【两者】
**人物状态**：
- 习近平：
- 拜登：
- 特朗普：
**事件摘要**：
**关键细节**：
**可信度**：【一手】/【二手】/【推测】
**来源**：
**关联条目**：
```

### 人物档案条目（KB02）
```markdown
### [维度名称]
**内容**：
**可信度**：【一手】/【二手】/【推测】
**来源**：
**关联时间轴**：
**关联象征条目**：
**最后更新**：
```

### 象征话语条目（KB03）
```markdown
### [关键词/话语]
**原文**：
**出处**：
**使用场景**：
**具身化方向**：【个人→国家】/【国家→个人】/【双向】
**关联档案**：
**可信度**：【一手】/【二手】
```

## 新增条目流程

```
Step 1: 检查是否与现有条目重复或矛盾
        使用kb-search确认
Step 2: 若有重复，报告给作者决定
Step 3: 若无重复，按格式规范创建条目
Step 4: 标注可信度和来源
Step 5: 检查是否需要更新关联条目
Step 6: 在KB05-meta-log.md记录新增操作
```

```
【新增条目报告】
目标文件：
条目ID/标题：
可信度：
来源：
关联条目更新：
```

## 修改条目流程

```
Step 1: 找到目标条目
Step 2: 在条目内保留原始内容（注释形式）：
        <!-- 原内容：[原文] 修改日期：[日期] 修改原因：[原因] -->
Step 3: 写入新内容
Step 4: 更新来源和可信度
Step 5: 检查关联条目是否需要同步更新
Step 6: 在KB05-meta-log.md记录修改操作
```

```
【修改条目报告】
目标文件：
条目ID/标题：
修改内容摘要：
修改原因：
关联条目更新：
```

## 删除条目流程

```
Step 1: 确认作者已批准删除
Step 2: 检查该条目是否被其他文件引用
Step 3: 若有引用，列出所有引用位置
Step 4: 获作者最终确认
Step 5: 执行删除
Step 6: 在KB05-meta-log.md记录删除操作
```

## 内部矛盾处理

发现两个条目相互矛盾时：
```
⚠️ 条目矛盾
条目A：[文件路径] ##[章节]
内容A：[摘要]
条目B：[文件路径] ##[章节]
内容B：[摘要]
矛盾性质：[描述]
可能的解决方向：（列举，不裁定）
请作者决定以哪个为准。
```

## 知识库审计流程

```
Step 1: 扫描所有KB文件
Step 2: 检查时事类条目是否已过时
Step 3: 检查条目间是否存在矛盾
Step 4: 检查已发现但未填补的缺口
Step 5: 检查引用格式是否规范
Step 6: 输出审计报告
```

```
【知识库审计报告】
审计日期：
审计范围：
过时条目：
内部矛盾：
未填补缺口：
格式问题：
建议优先处理：
```

## 禁止事项
- 禁止删除任何文件而不获作者确认
- 禁止自行裁定矛盾条目的优先级
- 禁止修改/story/文件夹内的任何文件
- 禁止修改.agents/rules/内的任何文件
